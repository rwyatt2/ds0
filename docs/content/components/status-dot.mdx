## Overview

StatusDot is a compact visual indicator used to communicate the real-time status of an entity — such as a user, service, or device. It uses color and an optional pulse animation to convey state at a glance, making it ideal for chat applications, monitoring dashboards, and contact lists.

## Usage

<ComponentPreview>
  <StatusDot variant="online" label="Online" />
</ComponentPreview>

```tsx
import { StatusDot } from '@ds0/react/status-dot';

export function Example() {
  return <StatusDot variant="online" label="Online" />;
}
```

## Accessibility

### Keyboard Interactions

| Key | Action |
|---|---|
| N/A | Display-only component, not interactive |

### Screen Reader

Announces the status label when encountered in reading order. Use the `label` prop to provide a descriptive accessible name.

### ARIA

- `role="status"` — live region for status updates
- `aria-label` — descriptive status text (falls back to variant name)

## API Reference

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `variant` | `'online' \| 'offline' \| 'busy' \| 'away' \| 'error' \| 'warning' \| 'neutral'` | `'neutral'` | Status variant controlling color |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Size of the indicator |
| `pulse` | `boolean` | `false` | Whether to show pulse animation |
| `label` | `string` | — | Accessible label for screen readers |
| `className` | `string` | — | Additional CSS classes |

## AI Decision Guide

**Use StatusDot when:**
- Showing online/offline/busy status for users in a contact list or chat
- Indicating service health in a monitoring dashboard
- Displaying device connectivity status

**Don't use StatusDot when:**
- You need to show progress toward a goal → use Progress instead
- You need a text label with semantic color → use Badge instead
- You need to communicate a detailed message → use Alert instead

## Related Components

- **Badge** — Use when you need text content alongside the indicator
- **Avatar** — Commonly composed with Avatar to show user presence
- **Spinner** — Use for indeterminate loading, not status indication
